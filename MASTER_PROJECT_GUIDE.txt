1. PROJECT THEME & VISION
-------------------------
This project is an end-to-end security solution designed to mitigate the risks 
of phishing and credential theft. It moves away from "static" security (simple 
blacklists) towards "Active Intelligence." By utilizing Machine Learning, 
Heuristic Scanners, and Dynamic Web Scraping, it identifies threats that 
standard antivirus software often misses.

2. CORE MODULES & FEATURES
--------------------------
LAYER 1: MACHINE LEARNING (The Static Brain)
   - Predicts malicious intent based on 30+ URL features.
   - Uses Random Forest models trained on 11,000+ data points.
LAYER 2: HEURISTIC ANALYSIS (The Pattern Brain)
   - Detects Homograph attacks (identical-looking Unicode characters).
   - Detects Typosquatting (brand impersonation).
   - Flags High-Risk TLDs (.xyz, .top, .ru, .cn).
LAYER 3: DYNAMIC SCANNED INTELLIGENCE (The Investigator)
   - Cross-Origin Auditor: Detects forms sending data to external domains.
   - Open Redirect Scanner: Detects malicious "Tunneling" in URLs.
   - Subdomain Takeover: Checks DNS CNAMEs for abandoned cloud buckets.
   - Script Auditor: Flags suspicious third-party JavaScript.

3. COMPREHENSIVE Q&A: 100 FREQUENTLY ASKED QUESTIONS
================================================================================

Q1: What is the main purpose of this project?
A1: To provide a high-end, research-driven platform for detecting phishing 
    at the network and infrastructure level.

Q2: What is the primary tech stack?
A2: Python (FastAPI), MongoDB (Database), and Scikit-learn (Machine Learning).

Q3: How does the ML layer work?
A3: It analyzes the structure of a URL and compares it against statistical 
    patterns found in thousands of known malicious links.

Q4: What is a Homograph attack?
A4: An attack where characters from different languages (like Cyrillic 'а') 
    are used to visually mimic a real brand name.

Q5: How do we detect Homograph attacks?
A5: By scanning for the `xn--` Punycode prefix and looking for any non-ASCII 
    characters in the domain name.

Q6: What is Typosquatting?
A6: Registering domains that are slightly misspelled versions of popular brands 
    (e.g., `goog1e.com` instead of `google.com`).

Q7: What is an Open Redirect?
A7: A vulnerability where a legitimate domain is used to redirect a user 
    to an external malicious URL without their knowledge.

Q8: How does the system find Open Redirects?
A8: It scans URL parameters like `?url=`, `?next=`, and `?goto=` for links 
    pointing away from the home domain.

Q9: What is a Subdomain Takeover?
A9: When a subdomain (e.g., `dev.yoursite.com`) points to a dead cloud service 
    (like an expired AWS bucket) that an attacker can then claim.

Q10: How do we check for Subdomain Takeovers?
A10: By resolving the domain's CNAME record and checking if it points to a 
     "Dead Service" signature (e.g., `s3.amazonaws.com`).

Q11: What is Clickjacking?
A11: An attack where a user is tricked into clicking buttons on an invisible 
     layer placed over a legitimate website.

Q12: How do we prevent Clickjacking?
A12: By checking for the `X-Frame-Options` and `Content-Security-Policy` 
     HTTP headers.

Q13: Why is MongoDB used in this project?
A13: For real-time storage of malicious IP/Domain blacklists and search history.

Q14: What is the role of BeautifulSoup4?
A14: To scrape the HTML of target websites to find suspicious forms and iframes.

Q15: What is "Explainable AI" (XAI)?
A15: The system translates complex technical scores into simple, human-readable 
     security briefings.

Q16: What weight is given to a Homograph attack detection?
A16: +90 Risk points (High Risk).

Q17: What weight is given to an Open Redirect?
A17: +40 Risk points.

Q18: What weight is given to a Subdomain Takeover?
A18: +70 Risk points.

Q19: What is the significance of the .xyz TLD?
A19: It is frequently used by attackers because it is cheap and easily accessible.

Q20: How many data points was the ML model trained on?
A20: Over 11,000 samples of both legitimate and phishing URLs.

Q21: What is the "Risk Score"?
A21: A total value from 0-100 derived from both ML results and Heuristic flags.

Q22: Can I use this project for live network traffic?
A22: Yes, the FastAPI engine is designed to handle asynchronous search queries.

Q23: How do I run the project?
A23: Activate the virtual environment (.nsvenv) and run `python app.py`.

Q24: What port does the server run on?
A24: Port 8000.

Q25: What is the "Master Test Suite"?
A25: A script (`master_test_suite.py`) that runs 15+ automated tests on the logic.

Q26: What is a "Cross-Origin" form?
A26: A form on `Site A` that sends its data to `Server B` (often a hacker's server).

Q27: How do we detect suspicious scripts?
A27: By scanning `<script src="...">` tags for high-risk domains like .ru, .cn, or .top.

Q28: What is KNN Imputation?
A28: Replacing missing data by finding "neighbors" with similar features.

Q29: Why is standard antivirus often ineffective here?
A29: Standard antivirus usually relies on signatures; our tool analyzes behavior.

Q30: What is the future scope of this project?
A30: Adding Computer Vision and real-time email scanning.
