# ### Core Purpose:
# The project started as a pure Machine Learning pipeline designed to train a model 
# that could classify network traffic/URLs as "Phishing" or "Legitimate".
#
# ### The "Before" Architecture (Step-by-Step Process):
#
# The original system was a linear pipeline orchestrated by `TrainingPipeline` 
# (in `networksecurity/pipeline/training_pipeline.py`).
#
# #### Step 1: Data Ingestion
# - **Component:** `DataIngestion` (`networksecurity/component/data_ingestion.py`)
# - **Action:** 
#   1. Connected to MongoDB (`NetworkData` database).
#   2. Exported collection data as a DataFrame.
#   3. Saved raw data to a feature store.
#   4. Split data into **Train** and **Test** sets.
# - **Key Output:** `train.csv`, `test.csv`
#
# #### Step 2: Data Validation
# - **Component:** `DataValidation` (`networksecurity/component/data_validation.py`)
# - **Action:**
#   1. Checked if the data had the correct number of columns (based on schema).
#   2. **Drift Detection:** Used `ks_2samp` (Kolmogorov-Smirnov test) to check if the 
#      statistical properties of the test data were significantly different from the 
#      training data.
#   3. Generated a drift report.
# - **Key Output:** Validated data files, Drift report.
#
# #### Step 3: Data Transformation
# - **Component:** `DataTransformation` (`networksecurity/component/data_transformation.py`)
# - **Action:**
#   1. Handled missing values using `KNNImputer` (K-Nearest Neighbors).
#   2. Replaced target values (-1 for phishing) with standard labels (0/1).
#   3. Created a preprocessing pipeline object.
# - **Key Output:** Transformed Numpy arrays, `preprocessor.pkl` object.
#
# #### Step 4: Model Training
# - **Component:** `ModelTrainer` (`networksecurity/component/model_trainer.py`)
# - **Action:**
#   1. Trained multiple models:
#      - Random Forest
#      - Decision Tree
#      - Gradient Boosting
#      - Logistic Regression
#      - AdaBoost
#   2. **Hyperparameter Tuning:** Optimized parameters for each model.
#   3. **Evaluation:** Compared models based on **F1 Score**.
#   4. **Selection:** Picked the single best performing model.
#   5. **Tracking:** Logged experiments (metrics, params, artifacts) to **MLflow** (hosted on DagsHub).
# - **Key Output:** `model.pkl` (The final trained model).
#
# #### Step 5: Artifact Syncing
# - **Action:** Synced all generated artifacts (models, reports, data) to an **AWS S3** bucket for storage and versioning.

2. PROJECT CORE ARCHITECTURE (3-LAYER DEFENSE)
-----------------------------------------------

LAYER 1: MACHINE LEARNING (The Static Brain)
   - Predicts malicious intent based on 30+ URL features.
   - Uses Random Forest models trained on 11,000+ data points.
   - Saves model as `.pkl` file for real-time predictions.

LAYER 2: HEURISTIC ANALYSIS (The Pattern Brain)
   - Detects Homograph attacks (identical-looking Unicode characters).
   - Detects Typosquatting (brand impersonation).
   - Flags High-Risk TLDs (.xyz, .top, .ru, .cn).
   - Identifies Bot-evasive patterns (like '@' obfuscation).

LAYER 3: DYNAMIC RESEARCH SCANNERS (The Investigator)
   - Cross-Origin Auditor: Detects forms sending data to external domains.
   - Open Redirect Scanner: Detects malicious "Tunneling" in URLs.
   - Subdomain Takeover: Checks DNS CNAMEs for abandoned cloud buckets.
   - Behavioral Auditor: Scans for suspicious script origins (JS).

4. COMPREHENSIVE Q&A: 50 KEY QUESTIONS

Q1: What is Safe-Surf's main purpose?
A1: To provide a high-end, research-driven platform for detecting phishing

Q3: How does the "Homograph" detection work?
A3: It scans for the `xn--` Punycode prefix and searches for any non-ASCII 
    characters that visually mimic a known brand domain.

Q4: What is the "Layer 3" dynamic scanner?
A4: It's an automated investigative crawler that visits the site to audit

    intelligence that anyone can understand.

Q6: What is a "Cross-Origin Data Leak"?
A6: It is when a fake login form on `Site A` is programmed to send the 
    user's password directly to a different `Server B` owned by hackers.

Q7: How is the ML model trained?
A7: Through a modular pipeline (Ingestion -> Validation -> Transformation -> 
    Training) orchestrated by `training_pipeline.py`.
