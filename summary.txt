# Network Security Intelligence Tool - Project Summary

## What I Built

I developed a full-stack Network Security Intelligence Tool that detects phishing threats in real-time. The system combines database lookups with AI-powered heuristic analysis to identify malicious IPs, domains, and URLs.

## The Problem I Solved

Traditional phishing detection systems have two major limitations:
1. They can only detect threats that are already in their database
2. They don't provide a user-friendly interface for quick checks

My solution addresses both issues by creating a hybrid detection system with an intuitive web interface.

## Technical Architecture

### Backend (Python + FastAPI)
- Built RESTful API endpoints for search, training, and prediction
- Integrated MongoDB for storing millions of threat records with optimized indexes
- Implemented MLflow for machine learning experiment tracking
- Deployed using Docker containers with AWS ECR integration

### Database Design
I ingested 4 large CSV datasets into MongoDB:
- 23,909 malicious IP addresses
- 9,897 malicious domains
- Thousands of phishing URLs from multiple sources

Each collection has indexed fields for sub-second query performance.

### Frontend (HTML + Bootstrap + Jinja2)
- Created a clean, responsive web interface
- Implemented color-coded risk scoring (Green/Yellow/Red)
- Added real-time search with instant feedback

### Intelligent Threat Detection Engine

I developed a two-layer detection system:

**Layer 1: Database Lookup**
- Queries MongoDB collections based on input type (IP/Domain/URL)
- Assigns weighted scores based on which datasets contain the threat

**Layer 2: Heuristic Analysis**
This is the innovative part - I created pattern-matching algorithms that detect NEW threats even if they're not in the database:

1. **Typosquatting Detection**
   - Uses fuzzy string matching (SequenceMatcher algorithm)
   - Identifies fake versions of popular brands (e.g., "paypa1.com" vs "paypal.com")
   - 80%+ similarity threshold triggers warning

2. **Dot-Split Evasion Detection**
   - Catches domains that split brand names with dots (e.g., "ama.zon.com")
   - Reconstructs the name and checks against known brands

3. **Suspicious Pattern Recognition**
   - Scans for phishing keywords: "login", "verify", "secure", "account"
   - Flags high-risk TLDs: .xyz, .top, .ru, .club
   - Detects IP addresses masquerading as URLs

4. **Complexity Analysis**
   - Identifies unusually long URLs (>75 characters)
   - Calculates cumulative risk score

### Risk Scoring Algorithm

I designed a 0-100 point scoring system:
- Database matches: 20-50 points depending on dataset
- Heuristic flags: 10-50 points per detected pattern
- Final score is capped at 100 and displayed with visual indicators

## Key Features

### 1. Universal Search
Users can input any of the following:
- IP addresses (e.g., 192.168.1.1)
- Domain names (e.g., example.com)
- Full URLs (e.g., http://example.com/login)

The system auto-detects the input type and searches accordingly.

### 2. Real-Time Analysis
Results appear instantly with:
- Risk score percentage
- Threat level classification
- Detailed reasoning for the score
- Database sources where threat was found
- Heuristic warnings (if any)
